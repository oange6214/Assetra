# 新增交易對話框全面重構 — 設計文件

日期：2026-07-15｜狀態：已與使用者逐段核可（版面 mockup v2 + 架構 + VM 收斂 + 驗證/分波）

## 1. 目標與原則

- **交易簡單**：最常用情境（台股買入、IB 美金帳戶買美股）填 2–3 欄就能存；買/賣是第一級類型，其餘七種收進「更多」。
- **資料準確**：確認寫入層（`Confirm.*` dispatch、GROSS 語意、雙保險 FX、`WithSameCurrencyFxCleared`）**一行不動**；同樣輸入必須寫出 bit-identical 的結果。驗證**只嚴不鬆**——允許新增檢查，不允許放寬。
- 執行路線：**原地分波重構**（非並行新殼），每波獨立可出貨、綠燈才進下一波。

## 2. 已定案的產品決策

| 決策點 | 定案 |
|---|---|
| 範圍 | 全面重構：版面＋共用元件＋切換機制＋VM 收斂 |
| 寫入行為 | 不變；驗證只嚴不鬆 |
| 快樂路徑 | 極簡：數量＋價格（或總額）即可存；手續費自動、扣款帳戶記住上次、日期預設今天 |
| 常用類型 | 僅買入/賣出 → 類型選擇器第一排 chips；其餘收「更多 ▾」 |
| 跨幣別觸發 | **成交幣別 ≠ 扣款帳戶幣別** 才顯示結算卡。台股 TWD→TWD 與 IB USD→USD 都走極簡路徑（IB 的換匯是另一筆「轉帳」交易）；只有複委託（USD 成交 ← TWD 扣款）出結算卡 |
| 跨幣別預設 | 「依帳戶明細」（使用者輸入當下通常看得到券商實際扣款金額） |
| 複委託雜費 | **不拆欄**：單一「費用合計（含雜費）」欄＋實際扣款錨定現金流；不動 schema |

## 3. 版面設計（mockup v2 定案）

Mockup 存於 `.superpowers/brainstorm/1548-1784049328/content/buy-form-layout-v2.html`。

- **類型選擇器**：下拉 → chips（`買入` `賣出` `更多 ▾`）。「更多」為彈出選單，收其餘七種類型。
- **資產 chip 內含幣別**（例：`DRAM · USD`）；原「交易幣別」整列**刪除**。幣別語意沿用現行 `IsTxCurrencyEditable`：投資表單幣別由資產決定（唯讀、併入資產 chip）；現金/負債表單幣別跟隨所選帳戶（唯讀顯示於表單內）；若既有流程存在「無帳戶可選且需手選幣別」的情境，該表單內保留一顆小型幣別下拉，不回全域列。
- **成交明細區**：`每股價格｜成交總額` 切換移到**區塊標題旁**（Buy/Sell 同位置），脫離進階。
- **結算摘要列常駐**（綠底 strip）：同幣別顯示 `成交總額 · 費用(估) · 總成本`；跨幣別顯示 `成交價金（USD 成本基礎）`。存檔前一眼可核。
- **扣款帳戶一列**：checkbox 與帳戶下拉合併，預設帶上次使用的帳戶。
- **結算卡（SettlementCard）**：跨幣別時出現在**主流程**（不在進階、不再需要「自動展開進階」workaround）。◉ 依帳戶明細（預設，「實際扣款」含一切雜費）／○ 依匯率估算（自動帶入，可手改）。卡內含溢價檢查列（§5）。
- **進階 Expander** 只剩：費用合計（選填，含雜費）、手續費折扣。同幣別時預設收合。
- **說明文字收斂**：長段 helper 改 ⓘ tooltip；表單內僅保留一行內的關鍵提示。
- 賣出表單套用同一結構（欄位名不同：賣出數量、入帳帳戶、費用預覽含證交稅）。

## 4. 架構設計

### 4.1 表單切換：九張常駐 → 懶切換

- 對話框中段改為 `ContentControl`，以 **TxType 字串 → DataTemplate** 的 selector 選模板；僅當前類型的表單被實體化。
- 刪除 ~20 個 `TxTypeIsXxx` 布林與 `OnTxTypeChanged` 的手動通知清單。
- **切換不丟資料**：欄位值住在常駐 sub-VM（`Buy`/`Sell`/`Div`/`Loan`/`CreditCard`/`Transfer`），view 重建後綁回同一 VM。
- 新增類型的改動點從 4 處（XAML host／flags／通知清單／dispatch）降為 2 處（模板／dispatch）。

### 4.2 共用元件（抽出重複區塊）

| 新控制項 | 取代 | 現況重複 |
|---|---|---|
| `FormField`（label＋input＋error 模板） | 手寫 label/間距/`FormFieldError` | ~30 處 |
| `TxFeeField` | 費用欄＋error＋hint | 7 張表單 |
| `TxCashAccountPicker` | checkbox＋帳戶下拉＋「將新增」提示 | 4 張表單 |
| `SettlementCard`（由 `CrossCurrencyOverlay` 改造） | 結算模式卡＋新溢價列 | Buy/Sell/Div 共用（DataContext＋4 DP 機制沿用） |
| 自適應 footer | 桌面版＋窄版兩份重複 footer | 2 → 1 |

### 4.3 VM 收斂（風險最高）

**歸屬規則：經濟欄位歸 sub-VM；跨表單欄位歸對話框 VM。**

- `Buy.*`/`Sell.*`/`Div.*` 各自持有：數量、價格、總額、費用預覽、結算模式、匯率、實際扣款。
- 對話框 VM 持有：類型、日期、資產、備註、錯誤、扣款帳戶、費用合計。
- 簡單現金表單（收入/存提/信用卡/轉帳）維持共用 `TxAmount` 不動。

**Buy 遷移**：`BuyTxViewModel` 長出自己的 `Quantity`/`Price`/費用預覽；費用計算抽共用 helper（新增資產對話框與買入表單同一套）；確認路徑照樣組 `StockBuyRequest` → `ExecuteStockBuyAsync`，只是值來源改讀 `Buy.*`。**獨立的「新增資產」對話框（`AddAssetDialogViewModel`）不動。** Sell 的單價由共用 `TxAmount` 移入 `Sell.Price`，比照辦理。一次遷一張表單。

## 5. 驗證強化（只嚴不鬆，全在 UI/VM 層）

1. **總成本溢價檢查**（跨幣別，實作在 SettlementCard 的 VM）：
   - `溢價 % = (實際扣款 − 價金 × 市場匯率) ÷ (價金 × 市場匯率)`。
   - 市場匯率來源優先序：交易日 `fx_rate_history`（Yahoo）→ 即時匯率（CurrencyService）→ 都查無則**靜默跳過**（絕不因無匯率資料擋記帳）。查詢非同步，不阻塞輸入。
   - 分級（買入方向；賣出/股利檢查絕對偏差）：`≤3%` 綠 ✓（複委託正常費用）；`3–10%` 黃色警告不擋；`>10%` 確認鈕變「仍要儲存」需二次確認（不絕對封死）。
   - 明確取代 mockup v1 的「匯率偏離市價」檢查——那會對每筆正常複委託誤報（反推匯率吸收雜費，偏高 0.1–1% 屬正常）。
2. **同幣別 fx 防呆**：寫入層 `TradeSqliteRepository.BindTradeParams` 的 `WithSameCurrencyFxCleared` 守門保留，補特徵測試釘住。
3. 既有驗證全數保留：正數/非負、折扣 0.01–1.0、賣出數量 ≤ 持股、轉帳來源≠目標、跨幣別模式必填 gate。

## 6. 測試策略

- **Wave 0 特徵測試（先鎖行為再動結構）**：以「輸入 → 確認層請求/寫入參數」斷言，鎖住：
  - Buy 五情境：同幣別／跨幣別依明細（fx 反推 `= (cash−fee)/(price×qty)`）／跨幣別依匯率（cash 估算 `= price×qty×fx+fee`）／Mode-C 反推單價／總額含手續費（`TotalIncludesFee`）。
  - Sell：GROSS 總額模式（`price = TotalProceeds/qty`）＋跨幣別兩模式；Div：per-share/total ＋跨幣別。
  - 同幣別 fx=null 守門。
- 每波結束：全測試套件綠＋建置零警告；遷移類（Wave 2）要求特徵測試 bit-identical。
- 新增單元測試：溢價分級邏輯（含無匯率跳過）、SettlementCard VM、切換後 sub-VM 狀態保留。
- 每波之間使用者目視驗收一次。

## 8. 工程注意事項

- UI 字串沿用 `Portfolio.Tx.*`；新增 key（溢價訊息、費用合計標籤、更多選單）**zh-TW＋en-US 兩檔同步**。
- 「記住上次扣款帳戶」寫 AppSettings 時用 `raiseChanged:false`（避免全域刷新迴圈，先前踩過）。
- 溢價檢查的匯率查詢在背景執行緒完成後，更新 UI 綁定屬性即可（單屬性通知可跨執行緒）；**不得**在背景執行緒動任何 UI 綁定集合。
- 不引用 `Stockra.*`；DataGrid 欄標題慣例照舊。
- 建置於 worktree 進行（主 checkout 可能被 VS 鎖 DLL）。

## 9. 不做的事（YAGNI）

- 不做精靈式 step-by-step 流程。
- 不改九種類型的功能語意（只改曝光順序與版面）。
- 不動獨立的「新增資產」對話框流程。
