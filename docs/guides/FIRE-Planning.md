# FIRE Planning Guide

Assetra 的 FIRE 頁面用來估算「財務自由所需資產」、達成所需年數，以及退休後資產是否能支撐到指定壽命假設。

## 使用情境

FIRE 適合回答這幾個問題：

- 以目前 Assetra 的淨資產來看，距離財務自由還差多少？
- 年支出與安全提領率改變時，需要多少資產？
- 每年儲蓄與預期報酬率改變時，預估幾年後達成？
- 退休後年支出、通膨與壽命假設下，資產是否可能提早耗盡？
- 不同情境，例如保守、基準、積極，結果差多少？

## Basic Mode

Basic mode 保留最簡單的 FIRE number 計算：

- 目前淨資產
- 年支出
- 年儲蓄
- 預期年化報酬率
- 安全提領率

主要公式是：

```text
財務自由所需資產 = 年支出 / 安全提領率
```

畫面會顯示所需年數、預估達成年份、達成時淨資產與資產路徑。年支出與年儲蓄欄位下方會顯示每月平均，協助使用者確認輸入的是年度金額。

## 報酬率與通膨

Basic mode 的預期年化報酬率以實質報酬率解讀，也就是已經扣除通膨後的報酬。Advanced mode 會保留通膨率欄位，適合把退休後支出與提領路徑拆開觀察。

如果你手上的是名目報酬率，例如預估投資年化 7%、通膨 2%，請避免在 Basic mode 直接輸入 7% 後又另外用通膨修正結果。比較保守的做法是輸入約 5% 的實質報酬率，或在 Advanced mode 中明確管理通膨假設。

## Advanced Mode

Advanced mode 會把假設明確保存到情境中，包含：

- 目前年齡
- 預期壽命
- 退休後年支出
- 通膨率

這些欄位會用於退休提領路徑與風險提示。若目前年齡與預期壽命都有填，Assetra 會估算退休後每年的資產變化，並在資產可能提前耗盡時顯示警示。

## 情境管理

FIRE 頁面支援多情境：

- 新增情境
- 儲存情境
- 複製情境
- 設為預設
- 刪除情境

情境會保存使用者輸入的主要假設，適合建立「保守」、「基準」、「積極」等不同規劃。預設情境會在進入 FIRE 頁面時優先載入。

## 淨資產來源

目前淨資產可以由 Assetra 財務總覽提供，也可以手動輸入。使用 Assetra 淨資產時，FIRE 結果會與目前帳戶、投資、負債與其他資產資料保持一致。

若選擇投資群組作為 FIRE 來源，結果會以該群組的目前淨值作為起點，並在同步到財務目標時保留群組連結，避免把全資產與特定資產組合混在一起比較。

## 同步到財務目標

按下「同步到財務目標」會建立或更新 FIRE 類型的財務目標。同步時會保存目標名稱、財務自由所需資產、目前淨資產、預估達成日期，以及目前情境名稱。再次同步同一個 FIRE 目標時會更新既有目標，不會建立重複項目。

刪除 FIRE 情境只會刪除情境本身，不會靜默刪除已同步的財務目標。

## 解讀結果

- `財務自由所需資產`：由年支出與安全提領率計算出的目標資產。
- `所需年數`：以目前淨資產、年儲蓄與預期報酬率估算達成 FIRE 的時間。
- `達成時淨資產`：達成年份的預估資產。
- `退休成功率`：Monte Carlo 模擬結果，適合比較情境，不代表保證。
- `退休提領路徑`：Advanced mode 下，退休後每年提領後的資產變化。
- `退休提領警示`：當提領路徑在預期壽命前耗盡，或假設資料不足時，畫面會用警示提醒。

## 注意事項

- FIRE 是規劃工具，不是投資保證。
- 預期報酬率、通膨率與安全提領率都會大幅影響結果。
- 若使用名目報酬率搭配通膨率，請確認沒有重複扣除通膨。
- 退休後年支出可與目前年支出不同，例如退休後房貸結束、醫療費增加，或支出結構改變。
