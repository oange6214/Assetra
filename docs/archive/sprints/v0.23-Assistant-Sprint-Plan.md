> **歷史文件**：實際 shipped baseline 已於 v0.27.0 完成，且與本草案不同。現況為 rule-based query routing + hybrid OpenAI/Ollama/Null LLM fallback + grounded read-only tools + insight cards + hosted insight polling + conversation history + Markdown export。現在執行請以 [`AI-Financial-Assistant-Spec.md`](../../planning/AI-Financial-Assistant-Spec.md)、[`Remaining-Work-Inventory.md`](../../planning/Remaining-Work-Inventory.md) 與 changelog 為準；本檔只保留作為 sprint 設計歷史。

> **狀態**：歷史草案（superseded）
> **規劃日期**：2026-05-03（v0.22.0 release 後）
> **預估規模**：M~L（4 個 sub-sprint，1.5–2 個工作日／sub-sprint）
> **前置版本**：v0.22.0（多元資產、情境模擬、Wpf.Ui UX redesign 已 ship）

## 1. 範圍與原則

**做什麼**：使用者用自然語言問財務問題，LLM 透過受控的 read-only tool calls 取資料、回答時引用來源。

**不做**：
- ❌ 寫入 / mutate（不刪交易、不建帳戶、不改貸款）
- ❌ 投資建議（系統 prompt 明確拒絕「該不該買 XX？」）
- ❌ 預測股價（明確拒絕）
- ❌ Free-form prompt to raw DB（任何 query 必須走 tool catalog）

**核心原則**：
1. **Read-only first** — v0.23.0 全 tool 都是 query；mutate 待 v0.23.x 之後
2. **Tool-calling 強制** — 不 inline DB 內容進 prompt，LLM 必須透過 tool 拿資料
3. **使用者持有 API key** — 沿用既有 user-secrets / DPAPI 加密
4. **可離線降級** — API 失敗時 graceful fallback，不卡住 UI
5. **資料來源強制 cite** — LLM 回答必須帶 `[T1]` 等標記，UI 解析成可點連結

## 4. Sprint 拆解（4 個 sub-sprint）

### v0.23.0-A — Port LLM Infrastructure（基底）

**目標**：把 Stockra 的 0-domain-coupling LLM infra 完整 port 過來，build / test 通過。

**Tasks**：

- **D1 Port Core/Ai 模型**（79 LOC）
  - `IAgentTool` → `IAssistantTool`（rename，邏輯不變）
  - `LlmModels` → `Assetra.Core.Models.Assistant.*`（拆檔）
  - `OrchestratorChunk` → `Assetra.Core.Models.Assistant.OrchestratorChunk`
  - 全部命名空間從 `Stockra.Core.Ai` → `Assetra.Core.Assistant`

- **F1 Port Provider 5 兄弟**（670 LOC）
  - `Anthropic.SDK` 加入 `Directory.Packages.props`
  - 其他 SDK / package（OpenAI / Gemini / Ollama HTTP client）一併同步
  - Provider 內部命名空間 reroute

- **F2 Port Factory + Mutable wrapper**（94 LOC）
  - `LlmProviderFactory`（runtime provider 選擇）
  - `MutableLlmProvider`（heat-swap when settings change）

- **F3 Settings UI**
  - Settings → AI 助理 section：
    - Provider 下拉（Claude / OpenAI / Gemini / Ollama）
    - API key 輸入（`ui:PasswordBox`）
    - Model 下拉（依 provider 動態載入）
    - 測試連線按鈕
    - Daily token cap 輸入（預設 100k）
  - 設定持久化用 DPAPI 加密（沿用 Fugle / Sync 既有 ProtectedData API）

**測試**：
- Mock provider response → factory 回傳對的實作
- API key 缺失 → fallback to NullLlmProvider 不 crash
- ~10 筆

**驗收**：
- App 啟動不 break
- Settings UI 可輸入 API key + 測試連線
- Provider switch 後 LLM call 走新 provider

### v0.23.0-B — Tool Catalog + Context Engineering

**目標**：定義 read-only finance tool 集，加上 context engineering 治理。

**Tasks**：

- **D1 IAssistantToolRegistry**
  - `Register(IAssistantTool)` / `GetAll()` / `GetByName(string)`
  - JSON schema export → 給 LLM 看的 tool descriptors

- **F1 第一批 Tool 實作**（每個 ~80 LOC）
  - `GetPortfolioSummaryTool` — 投資組合總覽（總市值、總成本、未實現損益）
  - `ListRecentTradesTool` — 最近交易（預設 20 筆，paging by `offset`）
  - `GetBalanceSheetTool` — 資產負債表快照
  - `QueryAssetByNameTool` — 模糊搜尋資產名稱（fuzzy match）
  - `GetMonthlySpendingTool` — 月度支出（by category）

- **F2 進階 Tool**（v0.23.0-B 後段或 v0.23.1 視時間）
  - `GetGoalProgressTool` — Goals 進度
  - `GetFireProjectionTool` — FIRE 試算結果
  - `GetMonteCarloOutcomeTool` — Monte Carlo P10/P50/P90
  - `GetIncomeStatementTool` — 損益表

- **F3 Context Engineering**
  - `AssistantContextBuilder.Build(conversation)` →
    1. System prompt（zh-TW、role、limits、cite format）
    2. 最近 N turns 的 message（windowing）
    3. 超過 N turn 的舊段 LLM-summarize 壓縮
  - `AssistantTokenBudget`
    - 計算 input + output token
    - 達 daily cap → 暫停 + UI 提示
    - 預估成本（依 provider model 計價）

- **F4 System Prompt 草稿**
  ```
  你是 Assetra 個人財務助理。
  - 永遠以 zh-TW 回答（繁體中文）
  - 回答時引用資料：每個事實標 [T1] [T2]，UI 會解析成連結
  - 算術一律呼叫 tool，不要自己心算
  - 不給投資建議（拒絕「該不該買」「會漲嗎」）
  - 不預測未來價格 / 走勢
  - 工具只能 read-only — 你看不到「修改」「刪除」工具
  - 若資料不足，明確說「我需要更多資訊」，不要瞎掰
  ```

- **F5 Tool Result 截斷**
  - 每個 tool 預設 ≤ 1500 tokens
  - 超過 → tool 回傳 `{"truncated": true, "next_offset": 20}`
  - LLM 自行決定是否再叫一次

**測試**：
- Tool registry register / lookup
- Tool result 截斷 boundary
- Context windowing：保留最近 10 turn，第 11 turn 觸發 summarize
- Token budget：達 cap 拒絕 dispatch
- ~20 筆

### v0.23.0-C — Chat UI + Conversation Loop

**目標**：使用者可開 NavRail「AI 助理」→ 開始對話。

**Tasks**：

- **D1 Port AgentOrchestrator → AssistantConversationService**（200 LOC，改 80）
  - Tool dispatch loop：max 5 rounds、單 tool timeout 30s、總 timeout 90s
  - tool error → 回傳 `{"error": "..."}` 給 LLM 重試 max 2 次
  - 每輪呼叫前 check token budget

- **F1 Port AiChatView shell → AssistantView**（53 LOC + 150 XAML）
  - chat list（ItemsControl + virtualization）
  - input 區（`ui:TextBox` + Send `ui:Button`）
  - 換 Wpf.Ui 樣式（這次重寫機會）

- **F2 重寫 AssistantViewModel**（~300 LOC，比 Stockra 簡化）
  - Inject 只有：`ILlmProvider`、`AssistantConversationService`、`AssistantToolRegistry`、`ISnackbarService`
  - **不** inject Portfolio / Goals / Categories VM（這些走 tool 拿）
  - `ObservableCollection<ChatMessageViewModel> Messages`
  - `SendCommand` / `CancelCommand` / `ClearCommand`
  - Streaming：邊 LLM stream 邊更新 UI

- **F3 ChatMessageViewModel**（port，40 LOC）
  - User / Assistant / ToolUse / ToolResult 四種樣式
  - Cite link 解析：`[T1]` → 點擊 trigger event 回到 source（trade / position / goal）

- **F4 Suggestion Chips**
  - 啟動時顯示 4-6 個 starter prompt：
    - 「我這個月花最多在哪？」
    - 「我的退休準備可以撐到幾歲？」
    - 「我最近 30 天有哪些交易？」
    - 「我的 FIRE 試算結果是什麼？」

- **F5 NavRail entry**
  - 加 NavSection.Assistant
  - Symbol：`SymbolRegular.BotSparkle24`（fallback `Chat24`）
  - 語言 keys：`Nav.Assistant` / `Assistant.Title` 等

- **F6 In-memory conversation**（v0.23.0 暫不持久化）
  - 重啟 app → conversation 清空
  - v0.23.1 再加 SQLite `assistant_conversation` table

**測試**：
- Tool loop max 5 rounds 後 abort
- Streaming response 段段送上 UI
- Cancel 中途取消 loop
- ~15 筆

**驗收**：
- 點 NavRail「AI 助理」→ 看到 chat UI
- 問「我有多少現金」→ LLM 呼叫 `GetBalanceSheetTool` → 答出含 `[T1]` cite
- 點 cite → 跳到對應位置

### v0.23.0-D — Harness Engineering（治理層）

**目標**：金融助理的數字正確性、用量控制、可觀測性。

**Tasks**：

- **D1 Audit Log**
  - SQLite schema：
    ```sql
    assistant_audit (
      id TEXT PRIMARY KEY,
      conversation_id TEXT,
      timestamp TEXT,
      role TEXT,           -- system / user / assistant / tool
      content TEXT,        -- 或 tool result JSON
      tool_name TEXT,      -- nullable
      tokens_in INTEGER,
      tokens_out INTEGER,
      provider TEXT,
      model TEXT
    )
    ```
  - `AssistantAuditRepository.AppendAsync(...)`
  - 每次 LLM call 寫一筆

- **D2 Token Usage Indicator**
  - UI 即時顯示 conversation 累計 token + 預估成本
  - 達 80% daily cap → 黃色警示
  - 達 100% → 紅色 + 暫停發送

- **D3 Tool Loop Guards**（已在 Sub-C 實作，這裡補測試）
  - 5 round / 30s call / 90s total / 2 retry
  - Settings 可調（advanced section）

- **D4 Tool Whitelist 驗證**（structural guarantee）
  - LLM 試圖呼叫不在 registry 的 tool name → orchestrator 拒絕
  - 系統 prompt 不告知任何 mutate tool 存在

- **D5 Sensitive Field Masking**（v0.23.0 預設 off）
  - Settings 開關「遮罩敏感資料」
  - 開啟時：account number `xxxx-1234`、symbol 不變、金額不變
  - 從 tool 結果送回 LLM 前過一次過濾器

- **D6 Replay/Debug Trace**
  - Settings 開關「儲存對話 trace」（dev-only）
  - 開啟時：每次 LLM I/O dump 成 JSON 到 `%APPDATA%\Assetra\assistant-traces\{conversationId}.json`
  - bug report 可附上

- **D7 Cost Tracking**
  - 依 provider + model 計價表（hardcoded，每月手動更新）
  - UI 顯示「今日：$0.0123 USD（5/100 USD daily cap）」
  - Settings 可看月度累積

**測試**：
- Audit log round-trip
- Token cap 觸發暫停
- Whitelist 拒絕未註冊 tool
- Masking on/off 結果差異
- ~15 筆

## 附錄 A：System Prompt 完整草稿

```
你是 Assetra 個人財務助理。

# 角色
- 你的工作是幫使用者理解他們的財務狀況：投資組合、收支、目標進度、退休規劃。
- 你不是投資顧問，不給「該不該買」的建議。
- 你不預測未來股價或市場走勢。

# 語言
- 預設以正體中文（zh-TW）回答。
- 使用者用英文問，可以用英文回；但內部 reasoning 仍可用中文。

# 工具使用（強制）
- 任何涉及具體數字、交易、餘額、時間範圍的問題，必須呼叫對應 tool。
- 不要自己心算 — 即使是兩個數字相加，也透過 tool（如果有）。
- Tool 結果如果有 truncated=true，按需求 paginate 多次。

# 引用格式（強制）
- 每個事實性回答必須帶引用標記：[T1] [T2] 等。
- 編號對應你呼叫過的 tool result（按順序）。
- UI 會解析 [T1] 成可點連結，使用者可以追溯。
- 範例：「你 5 月花最多在餐飲，共 12,450 元 [T1]，主要來自 5/15 王品集團 4,800 元 [T2]。」

# 限制
- 你只有 read-only tools。沒有「修改交易」「建立貸款」這類工具。
- 如果使用者要你「幫我刪掉那筆交易」「改成 3000」這類請求，明確告知 v0.23.0
  尚未開放寫入，請使用者自己在交易記錄頁操作。

# 拒絕情境
- 投資建議 → 「我不能給投資建議。但我可以幫你看資料，例如 [T1] 顯示你目前 XX 配置...」
- 預測 → 「未來走勢無法預測。我可以幫你看歷史 [T1] 或情境模擬 [T2] 的結果。」
```

## 附錄 C：Anthropic API 預估成本

以 Claude Sonnet 4.6 計（2026-05 定價）：
- Input: $3 / 1M tokens
- Output: $15 / 1M tokens

典型對話一次：
- System prompt 800 tokens + history 2000 tokens + tool result 1500 tokens = **input ~4300 tokens**
- Tool loop 平均 2.5 round → input × 2.5 = ~10750 tokens
- Output 平均 600 tokens

每次對話成本 ≈ $0.032 + $0.009 = **$0.04 USD**

預設 daily cap 100k input + 50k output ≈ $1.05 USD/day = **$31.5 USD/month**（重度使用）。
輕度使用（10 對話/day）≈ $12 USD/month。

使用者可在 Settings 下調 cap 到 1/10。
