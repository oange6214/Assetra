# v1.0 GA Sprint Plan

**Status:** Planning
**Trigger:** Once feature freeze on v0.27 / v0.28 + acceptable test count
**Estimated duration:** 4–6 weeks for one engineer

## Scope

Ship Assetra v1.0 — production-ready Windows desktop release with proper installer, code signing, auto-update, user docs, and a regression-tested feature set.

## Phase 1: Pre-GA hardening (1 week)

| Task | Effort |
|---|---|
| **E2 — Manual regression sweep**: 4 critical flows × 2 currencies × 2 themes — Buy/Sell flow, Transfer (same + cross-currency), tax report (annual + multi-year + AMT trigger), cloud sync round-trip, AI assistant 4 query patterns | 3 days |
| **E2 supplement — e2e automation**: pick the 2–3 highest-leverage flows; either WinAppDriver script or Appium-style; aim for one CI run per flow | 2 days |
| **Settings schema migration v0 → v1** — see `Settings-Schema-Migration-v1.md` | 2 days |
| **Bug bash on findings from above** | 1 day buffer |

Acceptance: regression sweep passes, schema migration unit-tested, no Critical/High bugs open.

## Phase 2: Distribution infrastructure (1–2 weeks)

| Task | Effort |
|---|---|
| **E3 — MSIX packaging pipeline** — wrap `dotnet publish` output, sign manifest, embed app identity | 3 days |
| **Code signing** — purchase EV certificate (~$300/yr) or use existing org cert; integrate into pipeline; sign every release artefact | 2 days |
| **E4 — Auto-update infrastructure** — pick Velopack (cross-platform) over Squirrel (legacy WPF). Wire update channel to GitHub Releases. Add in-app "check for updates" command. Test rollback path. | 4 days |
| **Release dry run** — produce a v0.99-rc1 MSIX, install on a clean VM, verify auto-update from rc1 → rc2 | 1 day |

Acceptance: clean Windows VM can install signed MSIX, app reaches first-run, auto-update detects newer release in the channel.

## Phase 3: User-facing docs (1 week)

| Task | Effort |
|---|---|
| **E5 — User guide** — first-run walkthrough, account setup, recording transactions, tax module, AI assistant. Markdown + screenshots, hosted as GitHub Pages or in-app help | 3 days |
| **Updated screenshots** — every feature page, light + dark theme, zh-TW + en-US | 1 day |
| **E7 — Release notes for v1.0** — combine v0.22-v0.27 changelogs into one user-facing page | 1 day |
| **Migration FAQ** — what's preserved, what's reset, where settings live | half day |
| **Privacy & security page** — what data is local, what goes to LLM, how cloud sync encrypts | half day |

## Phase 4: Soft launch (1 week)

| Task | Effort |
|---|---|
| Tag `v1.0-rc1`, share with closed beta (5–10 users) | half day |
| Beta period — collect feedback, hot-patch as needed | 5–7 days |
| Final `v1.0.0` tag | half day |

## Out of scope for v1.0

These are all v1.x or v2.0 candidates, NOT GA blockers:

- LlmApiKey OS credential store migration (v1.1)
- AI Phase 4 action automation (v1.2 — needs security review)
- PWA / mobile / push (v2.0 — D區塊)
- Multi-asset extensions (private funds, options) (v2.0)
- Sync Argon2id upgrade (v1.2 — current PBKDF2 is acceptable)
- Conflict UI improvements (v1.1)

## Risks

| Risk | Mitigation |
|---|---|
| EV cert procurement takes 1–2 weeks | Start in parallel with Phase 1, not after |
| Auto-update first-rollout breaks for users still on v0.x without update channel | Document manual `v0.x → v1.0` install steps |
| Beta finds critical bug post-cert-signing | Reserve 2–3 days buffer for re-sign + re-release |
| MSIX signing fails on certain Windows versions | Test on Windows 10 1809+ AND Windows 11 22H2+ |

## Acceptance criteria for v1.0 GA

- [ ] Manual regression sweep passes 4 critical flows × full matrix
- [ ] Settings schema migration handles v0.27 JSON without data loss
- [ ] Signed MSIX installs cleanly on Windows 10 + Windows 11
- [ ] Auto-update upgrades a `v0.99-rc1` install to `v1.0.0`
- [ ] User guide covers all top-level NavSection pages
- [ ] No Critical / High bugs open
- [ ] Test count ≥ 1200; flake rate ≤ 0.5%
