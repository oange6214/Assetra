# Assetra 功能藍圖與優先級 Roadmap

> 本文件同時包含「現況摘要」與「目標功能藍圖」。
> 若某項功能標記為已完成，預設表示至少已有可用 MVP；不一定代表完整版本、完整分析層與匯出能力都已齊備。

## 一、現況總覽

### 已實作模組

| 模組 | 內容 |
|---|---|
| 儀表板 | 總覽淨資產、配置 |
| 配置分析 | 資產配置、再平衡建議 |
| 投資 | 股票部位、買賣、配息、每日快照 |
| 帳戶 | 多現金帳戶、緊急預備金健康度 |
| 負債 | 貸款、信用卡、槓桿健康度、攤還表 |
| 交易記錄 | 完整時間軸 |
| 提醒 | `AlertRule`（基礎） |
| 收支與預算 | 分類管理、月度預算、月結摘要（`MonthlyBudgetSummaryService`） |
| 訂閱 / 固定支出 | 週期管理與自動入帳（`RecurringTransactionScheduler`） |

## 二、產品目標

Assetra 的定位應逐步從「投資追蹤工具」擴展成：

- 個人資產負債總覽平台
- 收支與預算管理工具
- 財務目標追蹤工具
- 投資績效與風險分析工具
- 長期財務規劃平台

## 三、功能模組藍圖

### A. 收支管理（Budgeting）
- 月度預算（按分類：飲食、交通、娛樂、醫療等）
- 自動分類 / 標籤規則
- 預算 vs 實際差異追蹤
- 訂閱與固定支出週期管理（自動產生交易）
- 月底結算報告

### B. 財務目標（Goals）

> 現況：已完成的是 `Goals MVP`，包含目標建立、進度與期限顯示。
> 自動 funding rule、milestone、完整 application-layer service 切分仍未完成。
- 目標金額 + 期限（買房頭期款、退休、旅遊基金、子女教育）
- 自動換算每月應存金額
- 進度視覺化
- 自動指派來源帳戶
- 達成里程碑通知

### C. 淨資產趨勢

> 現況：已完成的是折線圖、預設區間切換與自訂日期範圍的 MVP。
> 堆疊圖與重大事件標註仍未完成。
- 月 / 季 / 年資產減負債曲線
- 類別佔比堆疊圖
- 重大事件標註（如買房、換工作）

> 備註：已有 snapshot 資料，主要缺視覺化與分析呈現。

### D. 投資績效分析
- XIRR（含現金流的內部報酬率）
- TWR / MWR（時間加權 / 金額加權）
- 個股 vs 大盤（如 0050、S&P 500）對比
- 損益歸因：價差 / 配息 / 匯差

### E. 風險與稅務
- 配息日曆：除息日、預估配息、複利再投資模擬
- 稅務追蹤：股利所得、二代健保補充保費、海外所得最低稅負
- 風險指標：波動度、最大回撤、Sharpe Ratio、相關性矩陣
- 集中度警示：單一個股 / 產業 / 國家超過閾值
- 報稅資料匯出（CSV / 申報軟體格式）

### F. 多元資產擴展
- 外幣 / FX：美股、ETF、外幣存款、匯率自動換算
- 加密貨幣：BTC / ETH 持倉、交易所匯入
- 不動產：市值估算、租金收入、房貸關聯
- 保險：壽險、醫療、車險年費、保單現金價值
- 退休專戶：勞退自提、勞保年資、IRA / 401K
- 實物資產：黃金、藝術品、收藏

### G. 自動化與智慧化
- 報價自動更新（已有 TWSE / TPEX，可加美股 / 港股 / 加密）
- 銀行 / 證券 CSV / Excel 匯入
- 信用卡帳單 OCR / PDF 解析
- 配息自動入帳（除息日觸發）
- 異常交易偵測（金額、頻率異常）
- AI 財務助理（自然語言查詢）

### H. 報告與輸出

> 現況：已完成的是報表子系統第一階段，包含月結報告、損益表、資產負債表、現金流量表，以及 PDF / CSV 匯出。
> 後續仍可擴充更多期間摘要、進階分析視圖與其他輸出格式。
- 月 / 季 / 年度財務報表 PDF
- 三大報表：損益表、資產負債表、現金流量表
- 自訂期間摘要
- 圖表匯出（PNG / SVG）

### I. 體驗與運維
- 加密本機備份 / 還原
- 雲端同步（OneDrive / Google Drive 加密同步）
- 家庭多帳本（夫妻共用）
- 行動版（iOS / Android 或 PWA）
- 推播通知（帳單到期、目標進度、配息入帳）

### J. 差異化加值
- 情境模擬：通膨、利率、薪資變動對淨資產 30 年影響
- 退休試算：年金折現、4% 提領法則、FIRE 計算機
- 再平衡時機提示：偏離閾值自動建議
- 稅損收割（Tax-loss harvesting）建議

### K. 資料治理與匯入

> 現況（v0.7.0）：CSV / Excel 匯入 MVP 完成，覆蓋台股 Top 5 銀行 / 券商。preview + dedupe + conflict resolution + apply 全鏈路可用；歷史紀錄、回滾、對帳尚未完成。
- 匯入欄位映射規則 ✅（以宣告式 `CsvParserConfigs` / `ExcelParserConfigs` 提供，新增格式只需改 config）
- 匯入去重與重複交易偵測 ✅（`ImportConflictDetector` + `ImportMatchKey`；銀行資料以日期 / 金額為主，券商資料另納入方向 / 數量 / 單價以避免不同成交誤判）
- 手動確認與修正流程 ✅（每列 Skip / Overwrite / Add anyway 下拉，搭配現金帳戶必選）
- 匯入歷史紀錄與回滾 ⏳ v0.8+
- 對帳機制（帳戶 / 信用卡 / 證券）⏳ v0.8+
- `ImportRule`（自動分類 / 自動套用備註）⏳ v0.8+
- PDF / OCR 匯入 ⏳ Phase 2

## 四、優先級規劃

## P0：最優先（下一階段核心）

### 1. 收支管理（Budgeting）  *(已完成 ~95%)*
原因：
- 直接補齊目前最大缺口
- 能與帳戶、信用卡、固定支出形成閉環
- 也是後續目標、報表、AI 分析的基礎

現況：分類管理、月度預算、月結摘要服務、訂閱/固定支出週期均已實作；剩月結報告完整 UI。

### 2. 財務目標（Goals）
原因：
- 能把使用者從「看資料」帶到「做規劃」
- 和收支、帳戶、投資都能串起來
- 是產品黏著度很高的功能

### 3. 淨資產趨勢  *(基礎完成，視覺化未做)*
原因：
- 現有已有 snapshot 基礎，投入小、價值高
- 很適合做成高辨識度的核心頁面
- 能快速提升產品成熟感

現況：`PortfolioDailySnapshot` / `PositionSnapshot` 與 `DashboardTabPanel` 已備；剩月/季/年切換、堆疊圖、重大事件標註。

### 4. 資料治理與匯入
原因：
- 後續所有收支 / 對帳 / OCR / 自動分類都依賴這層
- 如果這層沒先做好，功能會越做越亂

## 六、架構提醒

### 1. 收支管理應作為獨立子系統
不要只是補幾個交易類型，應有自己的：
- 分類模型
- 預算模型
- 固定週期規則
- 月結流程
- 對帳與匯入規則

### 2. 信用卡應同時參與兩套邏輯
- 負債模型
- 收支 / 預算 / 分類模型

不能只把信用卡當負債，不然之後預算功能會斷掉。

### 3. 報表、績效、風險應共用計算引擎
建議拆成：
- 計算層
- 視覺化層
- 匯出層

避免每個畫面各算一份。

### 4. 匯入治理要先做
如果未來要做：
- CSV
- Excel
- OCR
- PDF
- 銀行帳單
- 信用卡帳單

就一定要先有：
- 欄位映射
- 去重
- 對帳
- 回滾
- 匯入確認流程

## 七、總結

Assetra 現在已經有不錯的投資、帳戶、負債基礎。  
接下來最值得投入的方向不是再補零散功能，而是建立：

1. 收支與預算
2. 財務目標
3. 淨資產趨勢
4. 資料治理與匯入

這四塊會決定 Assetra 能不能從「投資管理工具」升級成真正完整的「個人財務管理平台」。
