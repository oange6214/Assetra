## 二、產品目標

Assetra 的定位應逐步從「投資追蹤工具」擴展成：

- 個人資產負債總覽平台
- 收支與預算管理工具
- 財務目標追蹤工具
- 投資績效與風險分析工具
- 長期財務規劃平台

## 三、功能模組藍圖

### A. 收支管理（Budgeting）
- 月度預算（按分類：飲食、交通、娛樂、醫療等）
- 自動分類 / 標籤規則
- 預算 vs 實際差異追蹤
- 訂閱與固定支出週期管理（自動產生交易）
- 月底結算報告

### B. 財務目標（Goals）

> 現況：完整子系統已於 v0.16.0 出貨（`GoalMilestone` / `GoalFundingRule` / `GoalPlanningService` / `GoalProgressQueryService`）。
- 目標金額 + 期限（買房頭期款、退休、旅遊基金、子女教育）
- 自動換算每月應存金額
- 進度視覺化
- 自動指派來源帳戶
- 達成里程碑通知

### C. 淨資產趨勢

> 現況：v0.17.0 已補上堆疊圖（依資產類別）與重大事件標註（`PortfolioEvent`）。
- 月 / 季 / 年資產減負債曲線
- 類別佔比堆疊圖
- 重大事件標註（如買房、換工作）

> 備註：已有 snapshot 資料，主要缺視覺化與分析呈現。

### D. 投資績效分析
- XIRR（含現金流的內部報酬率）
- TWR / MWR（時間加權 / 金額加權）
- 個股 vs 大盤（如 0050、S&P 500）對比
- 損益歸因：價差 / 配息 / 匯差

### E. 風險與稅務
- 配息日曆：除息日、預估配息、複利再投資模擬
- 稅務追蹤：股利所得、二代健保補充保費、海外所得最低稅負
- 風險指標：波動度、最大回撤、Sharpe Ratio、相關性矩陣
- 集中度警示：單一個股 / 產業 / 國家超過閾值
- 報稅資料匯出（CSV / 申報軟體格式）

### F. 多元資產擴展
- 外幣 / FX：美股、ETF、外幣存款、匯率自動換算
- 加密貨幣：BTC / ETH 持倉、交易所匯入
- 不動產：市值估算、租金收入、房貸關聯
- 保險：壽險、醫療、車險年費、保單現金價值
- 退休專戶：勞退自提、勞保年資、IRA / 401K
- 實物資產：黃金、藝術品、收藏

### G. 自動化與智慧化
- 報價自動更新（已有 TWSE / TPEX，可加美股 / 港股 / 加密）
- 銀行 / 證券 CSV / Excel 匯入
- 信用卡帳單 OCR / PDF 解析
- 配息自動入帳（除息日觸發）
- 異常交易偵測（金額、頻率異常）
- AI 財務助理（自然語言查詢）

### H. 報告與輸出

> 現況：已完成的是報表子系統第一階段，包含月結報告、損益表、資產負債表、現金流量表，以及 PDF / CSV 匯出。
> 後續仍可擴充更多期間摘要、進階分析視圖與其他輸出格式。
- 月 / 季 / 年度財務報表 PDF
- 三大報表：損益表、資產負債表、現金流量表
- 自訂期間摘要
- 圖表匯出（PNG / SVG）

### I. 體驗與運維
- 加密本機備份 / 還原
- 雲端同步（OneDrive / Google Drive 加密同步）
- 家庭多帳本（夫妻共用）
- 行動版（iOS / Android 或 PWA）
- 推播通知（帳單到期、目標進度、配息入帳）

### J. 差異化加值
- 情境模擬：通膨、利率、薪資變動對淨資產 30 年影響
- 退休試算：年金折現、4% 提領法則、FIRE 計算機
- 再平衡時機提示：偏離閾值自動建議
- 稅損收割（Tax-loss harvesting）建議

### K. 資料治理與匯入

> 現況（v0.19.0）：CSV / Excel / PDF / OCR 全鏈路就位，含批次歷史與回滾、對帳子系統、自動分類規則。
- 匯入欄位映射規則 ✅（v0.7.0：宣告式 `CsvParserConfigs` / `ExcelParserConfigs`）
- 匯入去重與重複交易偵測 ✅（v0.7.0：`ImportConflictDetector` + `ImportMatchKey`；銀行以日期 / 金額 / 交易對象為主，券商交易會納入買賣方向、數量與單價，避免同日同股同額的不同成交被誤判為重複）
- 手動確認與修正流程 ✅（v0.7.0：每列 Skip / Overwrite / Add anyway 下拉）
- 匯入歷史紀錄與回滾 ✅（v0.8.0：`ImportBatchHistory` + `ImportRollbackService`）
- 對帳機制 ✅（v0.9.0–v0.10.0：`ReconciliationSession`、Created / OverwrittenFromStatement）

## 六、架構提醒

### 1. 收支管理應作為獨立子系統
不要只是補幾個交易類型，應有自己的：
- 分類模型
- 預算模型
- 固定週期規則
- 月結流程
- 對帳與匯入規則

### 2. 信用卡應同時參與兩套邏輯
- 負債模型
- 收支 / 預算 / 分類模型

不能只把信用卡當負債，不然之後預算功能會斷掉。

### 3. 報表、績效、風險應共用計算引擎
建議拆成：
- 計算層
- 視覺化層
- 匯出層

避免每個畫面各算一份。

### 4. 匯入治理要先做
如果未來要做：
- CSV
- Excel
- OCR
- PDF
- 銀行帳單
- 信用卡帳單

就一定要先有：
- 欄位映射
- 去重
- 對帳
- 回滾
- 匯入確認流程
